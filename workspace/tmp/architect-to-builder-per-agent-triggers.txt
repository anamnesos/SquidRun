(ARCHITECT #28): James locked the final design. Rescope Phase 1 to match.

**Final shape:**

**1. Per-agent triggers — each agent owns their wake conditions.**
- Each agent defines their own trigger config file (e.g., workspace/agent-trading/oracle-triggers.json)
- Trigger config can include:
  - timer cadence (e.g., every 5min, 10min, 15min — agent's choice)
  - event-based wakes (level tests, breakouts, volatility spikes — agent's choice)
  - wake-conditions combining the above
- Supervisor reads each agent's trigger config + fires dispatch ONLY when that agent's trigger fires

**2. Cross-review, not strategy modification.**
- Agents can observe and comment on each other's results (equity curve, win rate, method performance)
- Agents can NOT modify each other's strategy, triggers, or style
- Sovereign strategy per agent

**What stays shared (auditability requirements from Oracle):**
- Portfolio file format
- Audit log writes on every mutation
- Response schema: required rationale + required stop declaration + optional time-stop
- Live price feed source
- Scoreboard aggregation (phase 2)

**Re-sized Phase 1:**
- Original scope (shared timer dispatch) + per-agent trigger config loader
- Estimate probably moves to ~3-4 hours instead of 2-3, acceptable

**Seed configs:**
- Default trigger config for each agent = simple timer (5min Oracle / 10min Architect / 15min Builder — or whatever sensible defaults)
- Each agent will edit their own config over time as they tune their style

Push back if any of this fights the auditability model. Otherwise start on phase 1 with per-agent triggers from the start.
