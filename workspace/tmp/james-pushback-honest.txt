Mostly aligned, two minor refinements worth flagging — not blockers, just edges.

**1. Valid exits should include macro/event breaker.**
Current rule says exit only on: stop hit, time-stop hit, OR explicit chart-based thesis invalidation. But what if mid-trade we get an unrelated event — Iran fires, BTC exchange halts, Powell comments — that wasn't in the entry thesis? That's not "chart invalidation," but it clearly warrants an exit. Proposed: add "material macro/event breaker" as a fourth valid-exit reason. Keeps discipline, handles the black-swan case.

**2. Hunt-buffer should scale with recent 5m ATR, not be static.**
We're using BTC $80-150, SOL $0.30-0.50, alts 0.5-1%. On volatile days hunts can spike 1.5-2x further. On calm days, tighter is fine. Better version: buffer = 1.5x recent 5m ATR past the obvious level. Gives real-market calibration instead of an empirical guess. Builder can do this in the buffered-stop-default pass.

**3. Giveback alerts — agreement with framing.**
You said kill them. Oracle + Builder + I all locked on "stop-loss = exit, peak-giveback = noise." My only note: the DATA behind the alert isn't useless — a 100%-giveback on a position where you haven't moved stop to breakeven is telling you the thesis failed. The FIX is already in Builder's spec: replace the emotional "you gave back 100%" with factual "stop $X is $Y away, peak was $Z, now $W." Same data, no panic framing. That's what's shipping.
