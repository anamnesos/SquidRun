(ARCHITECT #27): Oracle landed clean design constraints for Phase 1. These are AUTHORITATIVE — build to this schema exactly.

**Prompt payload (scheduler sends to each agent):**
- live prices (BTC/ETH/SOL minimum)
- recent 5m structure summary
- that agent's own open paper positions
- that agent's realized/unrealized PnL
- scoreboard snapshot (later phase — minimal for now)
- current wake reason (timer cycle / event trigger / manual)

**Response schema (infrastructure hard-enforces):**
- REQUIRED: explicit rationale (free text, not optional)
- REQUIRED: explicit stop declaration OR explicit no-stop declaration (one or the other must be present)
- OPTIONAL: time-stop field (if included, must be explicit with time/price)
- Action payload: hold/open/close/scale + symbol/side/size as applicable
- Every mutation WRITTEN TO AUDIT LOG (not just the portfolio file)

**What the schema explicitly does NOT enforce:**
- Strategy (framework vs pure momentum vs counter-trend — agent choice)
- Stop placement discipline (past-hunt-zone buffer — agent choice)
- Scale-out cadence (agent choice)

The goal: comp stays honest (no silent mutations, every action justified), styles stay free (whose method wins reveals the best style).

Reject malformed responses (missing rationale / missing stop-declaration / silent mutation attempt) with a clear error back to the agent — they get to retry with a valid response next cycle.
